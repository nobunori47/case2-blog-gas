# AIブログ記事自動生成システム

GAS（Google Apps Script）+ Claude APIを使い、スプレッドシートにキーワードを入力するだけでSEOブログ記事を自動生成するシステムです。

## 概要

- スプレッドシート上でキーワード／サブキーワードを管理
- メニューから記事生成を実行すると、Claude APIが記事タイトル・本文（HTML）・メタディスクリプションを生成
- 「設定」シートに登録したサンプル記事のトーン・構成を参考にして生成
- ステータス管理（未生成 → 生成中 → 生成済み／エラー）で進行状況が一目で分かる
- 生成した記事はWordPress REST APIへ下書き投稿できる設計です。本課題ではスプレッドシートへの出力までを実装・検証しています（送信payloadはLogger.logで確認済み）

## スプレッドシート構成

| 列 | 内容 |
|---|---|
| A | キーワード |
| B | サブキーワード |
| C | ステータス（未生成／生成中／生成済み／エラー） |
| D | タイトル |
| E | 記事本文（HTML） |
| F | メタディスクリプション |

「設定」シートのA1セルに、トーン・構成の参考にするサンプル記事を入れておきます。

## セットアップ

### 1. Claude APIキーの取得

[Anthropic Console](https://console.anthropic.com/) でAPIキーを発行します。

### 2. Script Propertiesの設定

GASエディタの「プロジェクトの設定」→「スクリプト プロパティ」で以下を登録します。

| プロパティ名 | 内容 | 必須 |
|---|---|---|
| `CLAUDE_API_KEY` | Claude APIキー | ◯ |
| `WP_BASE_URL` | WordPressのURL（例: `https://example.com`） | WordPress連携時のみ |
| `WP_USERNAME` | WordPressユーザー名 | WordPress連携時のみ |
| `WP_APP_PASSWORD` | WordPress Application Password | WordPress連携時のみ |

**注意：APIキーやパスワードをコード内に直接書き込まないこと。** コードを共有した瞬間に漏洩リスクが生まれます。必ずScript Propertiesを経由してください。

### 3. スクリプトの紐付け

1. スプレッドシートを開く
2. 拡張機能 → Apps Script
3. `Code.gs` の内容を貼り付け
4. 保存してスプレッドシートをリロード

## 使い方

1. スプレッドシートのA列・B列にキーワード／サブキーワードを入力
2. 生成したい行を選択
3. メニュー「記事生成」→「選択行の記事を生成」を実行
4. ステータスが「生成中」→「生成済み」に変わったら、D〜F列にタイトル・本文・メタディスクリプションが出力される

### WordPressに下書き送信する場合（任意機能）

1. 記事生成済みの行を選択
2. メニュー「記事生成」→「選択行をWordPressに下書き送信」を実行
3. 成功するとログにWordPressの記事IDとURLが出力される（status: `draft`のため自動公開はされない）

## トラブルシューティング

### 記事が生成されない／ステータスが「エラー」のまま

- A列にキーワードが入力されているか確認
- 実行ログ（表示 → 実行数）でエラー内容を確認

### 「CLAUDE_API_KEYがスクリプトプロパティに設定されていません」エラー

- Script PropertiesにCLAUDE_API_KEYが正しく登録されているか確認

### JSONパースエラー

- Claude APIの出力がJSON形式から外れている場合に発生
- プロンプト内に「HTML属性はシングルクォートを使う」「ダブルクォートを属性値に使わない」等の制約を明示することで解決済み
- 再発した場合は実行ログの「クリーニング後の文字列」を確認し、プロンプトの制約をさらに厳格化する

### 実行時間6分超過エラー

- GASは1回の実行が6分でタイムアウトする制約がある
- 一度に選択する行数を減らし、少数行ずつ実行する

### APIクォータ超過エラー

- Claude APIの利用上限に達している可能性。Anthropic Consoleで使用量を確認

### WordPressに下書きが反映されない

- `WP_BASE_URL` / `WP_USERNAME` / `WP_APP_PASSWORD` がScript Propertiesに正しく設定されているか確認
- Application Passwordの権限、REST APIが有効になっているか確認
- 実行ログのHTTPステータスコードとレスポンス内容を確認（201以外はエラー）

## 工夫したポイント

### GASの制約への対応

- Script PropertiesでAPIキーを管理
- ステータス管理で処理状況を可視化
- 実行時間6分制限を考慮し、選択行単位で実行

### プロンプト設計

- サンプル記事を参照してトーンを再現
- タイトル、本文、メタディスクリプションを一括生成
- JSON形式で出力させることでデータ加工を容易にした

### 品質向上

- 文字数2,000〜3,000字を指定
- H2構成を固定化
- メタディスクリプションを120字以内で生成
